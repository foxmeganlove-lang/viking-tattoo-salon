# Viking Tattoo Salon — демонстрационный шаблон лендинга для тату-студии

Это демонстрационный шаблон одностраничного сайта для тату-студии в скандинавской / нордической стилистике. Проект предназначен для портфолио веб-разработчика и для показа клиентам возможной структуры будущего сайта.

Проект не является сайтом реально существующего салона. Название, персонажи, изображения, контакты и бизнес-данные используются только для демонстрации визуального стиля и функционала.

## Что уже работает в демо

- Адаптивная структура лендинга: первый экран, блок о проекте, услуги, стили, галерея, мастера, FAQ, шаги и контактная секция.
- Интерактивный калькулятор стоимости на чистом JavaScript.
- Демо-форма заявки: данные не отправляются наружу, после отправки показывается нейтральное сообщение.
- FAQ-аккордеон с клавиатурно доступными кнопками.
- Галерея с подписями, доступными на touch-экранах без hover.
- Конфигурационный файл `config.js` для текстов, услуг, FAQ и карточек мастеров.
- Минимальная schema.org-разметка для демонстрационной веб-страницы, без фейковых LocalBusiness / TattooParlor данных.

## Что заменить или подключить для реального клиента

- Реальные контакты: телефон, email, адрес, график работы, карта и ссылки на соцсети.
- Настоящие фотографии студии, мастеров и работ.
- Реальные имена мастеров, специализации, цены, условия записи и FAQ.
- Юридические тексты: политика конфиденциальности, согласие на обработку данных, правила записи и возрастные ограничения.
- Рабочую отправку формы: Telegram, почту, CRM, Formspree, Web3Forms, backend endpoint или другой согласованный канал.
- Production SEO: `canonical`, `og:url` и абсолютный `og:image` нужно заполнять только после публикации на реальном домене или GitHub Pages. В демо они не подставляются, чтобы не имитировать реальный бизнес.

## Демонстрационные данные

- Все изображения в `images/` являются демонстрационными AI-визуализациями и не представляют реальные фотографии студии или работы мастеров.
- Контактные данные в демо намеренно отключены. Контактный блок показывает, где будут располагаться реальные данные клиента.
- Соцсети, карта, `tel:`, `mailto:`, WhatsApp и Telegram-ссылки не используются, пока проект не адаптирован под реального клиента.
- Форма работает в `FORM_MODE: 'demo'` и не отправляет данные наружу.

## Кастомизация

Основная настройка выполняется в `config.js`:

- `STUDIO_NAME`, `STUDIO_SLOGAN`, `STUDIO_DESCRIPTION` — название и общие тексты.
- `CONTACT_NOTICE`, `PHONE`, `EMAIL`, `ADDRESS`, `HOURS`, `MAP_LINK` — контактный блок.
- `TELEGRAM_URL`, `WHATSAPP_URL`, `VK_URL`, `INSTAGRAM_URL` — соцсети и мессенджеры.
- `ADVANTAGES`, `SERVICES`, `MASTERS`, `FAQ` — редактируемые списки для блоков сайта.
- `FORM_MODE`, `FORM_ENDPOINT` — режим формы и endpoint для реальной отправки.

Для реального запуска заполните значения клиента и переключайте `FORM_MODE` на `production` только после настройки безопасного endpoint. Не храните токены Telegram-ботов и другие секреты в клиентском JavaScript.

## Изображения

Текущие файлы в `images/`:

- `hero_bg.webp` — фон первого экрана.
- `studio.webp` — демонстрационный интерьер.
- `master_ragnar.webp`, `master_lagerta.webp`, `master_bjorn.webp` — демонстрационные портреты.
- `sleeve.webp`, `odin.webp`, `fenrir.webp` — демонстрационные изображения для галереи.

При адаптации под клиента замените эти файлы на реальные изображения или обновите пути в HTML / `config.js`.

## Локальный запуск

Сайт статический и не требует установки зависимостей. Достаточно запустить любой локальный статический сервер из корня проекта, например:

```powershell
python -m http.server 8000 --bind 127.0.0.1
```

После запуска откройте `http://127.0.0.1:8000/`.

## Публикация

Для GitHub Pages достаточно разместить файлы проекта в корне репозитория и включить Pages для нужной ветки. После появления публичного URL заполните production SEO-поля: `canonical`, `og:url` и абсолютный `og:image`, если они нужны для конкретной публикации.
