# Audit Report

Дата проверки: 2026-07-05

## 1. Исправленные проблемы

- Сайт переведен в честную демо-подачу: добавлена маркировка "Демонстрационный концепт сайта для тату-студии" в demo badge и первом экране.
- Удалена `LocalBusiness` / `TattooParlor` JSON-LD-разметка с вымышленным адресом и телефоном. Добавлена минимальная schema.org-разметка `WebPage` для демонстрационной страницы.
- Убраны пустые `canonical` и `og:url`; README поясняет, что production URL нужно заполнять только после публикации на реальном домене или GitHub Pages.
- Галерея и футер теперь прямо указывают, что изображения являются демонстрационными и не являются работами реальных мастеров.
- Контактный блок сохранен визуально, но телефон, email, адрес, карта, Telegram, WhatsApp и VK заменены на демо-пояснение и не кликабельны.
- Неподтвержденные claims вроде "3000+ клиентов", "100% стерильность" и "пожизненный уход" заменены на нейтральные демонстрационные формулировки.
- Демо-форма не отправляет данные наружу и показывает нейтральное сообщение без вставки пользовательского ввода через `innerHTML`.
- Чекбокс формы больше не отмечен по умолчанию; текст заменен на понятное подтверждение демо-режима.
- Мобильное меню блокирует прокрутку body через `.no-scroll`, использует `100dvh`, `overflow-y: auto` и корректно обновляет `aria-expanded`.
- FAQ рендерится кнопками с `aria-expanded`.
- Опции калькулятора заменены на `button` с `aria-pressed`.
- Подписи карточек галереи доступны на touch-экранах без hover.
- Папка `admin` удалена: она не использовалась, а `admin/config.yml` отсутствовал.
- README переписан как документация демонстрационного шаблона, а не как описание готового реального бизнеса.

## 2. Реально проверенные сценарии в браузере

Локальный запуск выполнен без установки зависимостей через статический сервер на `http://localhost:8000/`.

Проверены viewport:

- 1440x1000: горизонтальной прокрутки нет, demo badge и hero note видны, schema type `WebPage`, чекбокс не отмечен по умолчанию.
- 768x900: горизонтальной прокрутки нет, галерея показывает подписи без hover, контактные демо-элементы не имеют href.
- 375x812: горизонтальной прокрутки нет, мобильная шапка и первый экран не обрезают текст по ширине.

Интерактивные проверки:

- Мобильное меню открывается и закрывается; при открытии `body.no-scroll` активен, `aria-expanded=true`, прокрутка body заблокирована.
- FAQ раскрывается кнопкой; первый пункт получает `aria-expanded=true`.
- Калькулятор реагирует на выбор "Реализм / Портрет"; цена меняется на `12 000`, выбранная кнопка получает `aria-pressed=true`.
- Пустая форма показывает ошибки валидации.
- После заполнения тестовыми данными и подтверждения демо-чекбокса появляется сообщение: "Демо-форма: заявка не отправляется. В версии для клиента подключаются Telegram, почта или CRM."
- Console errors: JavaScript-ошибок не обнаружено.
- Изображения: после прокрутки все 7 изображений загрузились, битых изображений нет.

## 3. Что сознательно осталось демо-элементом

- Бренд `VALHALLA INK` и вся студийная легенда являются вымышленной оболочкой для демонстрации визуального направления.
- Изображения в `images/` являются демонстрационными AI-визуализациями.
- Мастера, услуги, цены, FAQ, шаги работы и калькулятор показывают структуру будущего сайта, а не реальные условия конкретной студии.
- Форма остается в `FORM_MODE: 'demo'` и не отправляет данные наружу.
- Контактный блок, карта и соцсети сохранены как визуальные места под будущую интеграцию.

## 4. Что заменить при адаптации под реального клиента

- Название, оффер, описание студии и юридические тексты.
- Реальные фотографии студии, мастеров и выполненных работ.
- Реальные имена мастеров, роли, биографии и ссылки на соцсети.
- Телефон, email, адрес, график работы, карту и мессенджеры.
- Тарифы, коэффициенты калькулятора, услуги и условия записи.
- Реальный endpoint формы: Telegram, почта, CRM, Formspree, Web3Forms или backend.
- Production SEO: `canonical`, `og:url`, абсолютный `og:image`, title/description под опубликованный домен.
- При необходимости CMS-админку нужно добавлять отдельно вместе с рабочим конфигом.
